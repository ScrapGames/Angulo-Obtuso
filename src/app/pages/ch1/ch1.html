¿Qué es git?

Git es un sistema de control de versiones creado por Linus Torvalds en 2005 para gestionar el desarrollo del kernel de Linux.

Este permite a los desarrolladores rastrear cambios en el código fuente durante el desarrollo de software, siendo extremadamente útil para el trabajo en equipo, permitiendo a grupos enteros de programadores trabajar en distintas partes de un mismo proyecto manteniendo constantemete una traza de los cambios que se han realizado en el proyecto.

Puede llegar a ser confuso y duro de aprender para los usuarios novatos y/o poco experimentados en el mundo del software —incluidos quienes escriben este tutorial xd—, ya que se maneja principalmente desde la shell del sistema operativo (bash, zsh, powershell) a través de comandos. Si bien existen utilidades para manjear git desde interfaces gráficas, hay muchas cosas que han de hacerse si o si desde la shell.

Una de sus grandes virtudes es su velocidad, ya que como hemos mencionado anteriormente, fue diseñado para mantener una trazabilidad del kernel Linux, por lo que este debía de estar preparado para analizar, comparar o almacenar cientos de miles de líneas de código de manera eficiente.

Git esta distribuido bajo la licencia GPL y su código fuente puede ser encontrado en su repositorio de GitHub, aunque cabe resaltar que este repositorio es solo un mirror.
